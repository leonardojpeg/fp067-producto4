1. Cloud Functions - Envío de Push Notifications

1.1 Descripción

Se han implementado dos Cloud Functions en Firebase que se disparan automáticamente cuando se producen cambios en la Realtime Database:

  • onNewPlayer (onCreate): Se ejecuta cuando se añade un nuevo jugador a la base de datos. Envía una notificación push a todos los dispositivos suscritos al topic 'all'.
  • onUpdatePlayer (onUpdate): Se ejecuta cuando se modifica un jugador existente. Envía una notificación push informando del cambio.

2. Notificación en App Móvil - React Native

2.1 Descripción

La app móvil React Native ha sido configurada para recibir notificaciones push mediante @react-native-firebase/messaging. El código en App.js implementa:

  • Solicitud de permisos de notificación (Android 13+)
  • Obtención del token FCM del dispositivo
  • Suscripción al topic 'all'
  • Listener para notificaciones en foreground (Alert.alert)
  • Handler para notificaciones en background
  • Handler para apertura de app desde notificación

2.2 Configuración Android

Archivos configurados:
  • android/app/google-services.json — Configuración Firebase para Android
  • android/app/src/main/AndroidManifest.xml — Permiso POST_NOTIFICATIONS + canal FCM
  • package.json — Dependencias @react-native-firebase/app y @react-native-firebase/messaging

3. Notificación en App Web - Angular

3.1 Descripción

La app web Angular ha sido configurada para recibir notificaciones push mediante Firebase Cloud Messaging. Se implementaron:

  • NotificationService — Solicita permisos del navegador, obtiene token FCM con VAPID key, escucha mensajes en foreground
  • firebase-messaging-sw.js — Service Worker para recibir notificaciones cuando la app está en background
  • app.config.ts — Registro de provideMessaging() en la configuración de Angular
  • app.ts — Integración del servicio en ngOnInit

4. Justificación - Flujo Completo

BD Realtime Database (añadir/modificar jugador)
↓
Cloud Function se dispara (onCreate / onUpdate)
↓
Firebase Cloud Messaging envía push al topic 'all'
↓
┌─────────────────────────────────┐
│ │
↓ ↓
React Native Angular
(@react-native-firebase) (Service Worker + NotificationService)
↓ ↓
Alert.alert() en foreground Notification API del navegador
Notificación sistema en BG showNotification() en background

El flujo funciona de la siguiente manera:

  1. Un usuario añade o modifica un jugador en la Realtime Database de Firebase (ya sea desde la consola o desde la app Angular).
  2. La Cloud Function correspondiente (onNewPlayer o onUpdatePlayer) se dispara automáticamente.
  3. La función construye un payload con el título y cuerpo de la notificación, y lo envía al topic 'all' mediante admin.messaging().sendToTopic().
  4. Firebase Cloud Messaging (FCM) distribuye la notificación a todos los dispositivos suscritos al topic.
  5. En React Native: si la app está en foreground, se muestra un Alert.alert(). Si está en background, el sistema operativo muestra la notificación.
  6. En Angular: si la app está en foreground, el NotificationService recibe el mensaje y muestra una notificación. Si está en background, el Service Worker (firebase-messaging-sw.js) la maneja.
